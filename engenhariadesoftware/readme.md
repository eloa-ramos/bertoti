A engenharia de software aplica princípios de engenharia no desenvolvimento de software, focando em criar sistemas eficientes e de alta qualidade.
Diferente da programação, que é a escrita de código, a engenharia de software envolve um processo mais amplo e sistemático, utilizando conhecimentos
teóricos. Já a ciência da computação se dedica mais aos fundamentos teóricos, enquanto a engenharia de software usa esses conceitos para soluções
práticas. O Google, com sua experiência, pode contribuir para tornar o desenvolvimento de software mais rigoroso e confiável, à medida que a
tecnologia se torna mais essencial na vida cotidiana.

A engenharia de software vai além de escrever código, incluindo ferramentas e processos para manter o código ao longo do tempo. O livro propõe que a
engenharia de software é "programação integrada ao longo do tempo", focando em práticas para garantir que o código seja sustentável e adaptável.
Possuindo três princípios fundamentais que são:
1. Tempo e Mudança: Adaptar o código ao longo de sua vida.
2. Escala e Crescimento: Adaptar a organização conforme ela evolui.
3. Compromissos e Custos: Tomar decisões equilibrando esses fatores.
Eles visam tornar o código mais durável e a engenharia de software mais exigente.
